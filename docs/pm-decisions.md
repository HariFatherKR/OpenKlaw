### 2.2 이메일 (Outlook/Gmail)

| 항목 | 분석 |
|------|------|
| **한국 사무직 활용도** | ⭐⭐⭐⭐⭐ (필수) |
| **일일 사용 빈도** | 30-50건/일 (읽기/쓰기) |
| **주요 사용 시나리오** | 업무 요청, 보고, 일정 조율, 공식 커뮤니케이션 |
| **AI 자동화 기회** | 초안 작성, 톤 조정, 번역, 요약 |

| **경쟁사 지원** | 일부 (하지만 한국 비즈니스 격식 X) |

**결론:** P0 - 높은 수요, 낮은 구현 복잡도

---

### 2.3 엑셀 (Microsoft Excel / Google Sheets)

| 항목 | 분석 |
|------|------|
| **한국 사무직 활용도** | ⭐⭐⭐⭐ (높음) |
| **일일 사용 빈도** | 주 3-5시간 (데이터 작업) |
| **주요 사용 시나리오** | 데이터 정리, 보고서 수치, 피벗테이블, 차트 |
| **AI 자동화 기회** | 수식 생성, 데이터 분석, 차트 제안, CSV 요약 |
| **구현 난이도** | 중 (파싱은 쉬움, 수식 생성은 복잡) |
| **경쟁사 지원** | 제한적 |

**주요 Pain Point:**
- "수식이 복잡해지면 막힘"
- "여러 파일 데이터 합치기 귀찮음"
- "피벗테이블 매번 새로 만들기"

**결론:** P1 - MVP 이후 Phase 2에서 구현

---

### 2.4 캘린더 (Google Calendar / Outlook Calendar)

| 항목 | 분석 |
|------|------|
| **한국 사무직 활용도** | ⭐⭐⭐⭐ (높음) |
| **일일 사용 빈도** | 회의 3-5건/일 평균 |
| **주요 사용 시나리오** | 회의 일정, 일정 조율, 리마인더 |
| **AI 자동화 기회** | 일정 충돌 감지, 최적 시간 제안, 회의 준비 알림 |
| **구현 난이도** | 상 (OAuth 연동 필요, API 접근) |
| **경쟁사 지원** | Jan.ai Connectors (복잡) |

**주요 Pain Point:**
- "회의 시간 조정 메일 왔다갔다"
- "오늘 뭐 있지? 매번 확인"
- "회의 전 자료 준비 까먹음"

**MVP 제외 이유:**
- OAuth 연동 필요 → 보안 우려 증가
- 로컬 AI 철학과 충돌 (외부 API 호출)
- 기업 방화벽에서 차단 가능성

**결론:** P2 - 장기 로드맵 (v2.0+)

---

### 2.5 카카오톡 (메신저)

| 항목 | 분석 |
|------|------|
| **한국 사무직 활용도** | ⭐⭐⭐⭐⭐ (매우 높음) |
| **일일 사용 빈도** | 수십~수백 건 메시지 |
| **주요 사용 시나리오** | 팀 소통, 빠른 공유, 비공식 업무 요청 |
| **AI 자동화 기회** | 대화 요약, 답장 제안, 일정 추출 |
| **구현 난이도** | 최상 (공식 API 없음, 법적 리스크) |
| **경쟁사 지원** | ❌ 전무 |

**현실적 제약:**
1. **공식 API 없음** - 카카오톡 비즈니스 API는 기업용만
2. **개인 채팅 접근 불가** - 법적 이슈 (통신비밀보호법)
3. **스크래핑 금지** - 이용약관 위반
4. **보안 우려** - 민감한 대화 내용

**대안적 접근 (가능한 것):**
- 카카오톡 대화 내보내기 → 텍스트 요약 (수동)
- 카카오워크 API 연동 (기업용)

**결론:** P3 또는 제외 - 기술적/법적 제약 심각

## 6. Go/No-Go 기준

### 출시 Go 기준 (모두 충족 필요)

| # | 기준 | 측정 방법 | 목표 |
|---|------|----------|------|
| 1 | 설치 성공률 | 테스트 10명 중 | ≥ 80% |
| 2 | 첫 대화까지 시간 | 설치 시작부터 | ≤ 5분 |
| 3 | HWP 요약 성공률 | 테스트 문서 10개 | ≥ 90% |
| 4 | 이메일 생성 품질 | 사용자 평가 5점 만점 | ≥ 4.0 |
| 5 | 앱 안정성 | 1시간 사용 중 크래시 | 0회 |
| 6 | 첫 응답 속도 | 첫 토큰 출력까지 | ≤ 5초 |

### 출시 블로커 (하나라도 해당 시 No-Go)

| # | 블로커 | 설명 |
|---|--------|------|
| 1 | 설치 실패율 > 30% | 진입장벽 너무 높음 |
| 2 | HWP 파싱 실패 | 핵심 차별점 미동작 |
| 3 | 데이터 유출 가능성 | 로컬 철학 위반 |
| 4 | 응답 속도 > 30초 | 실사용 불가 |
| 5 | 치명적 버그 미해결 | 데이터 손실 등 |
