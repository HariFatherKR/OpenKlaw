## 2. 상세 경쟁 분석

### 2.1 Ollama

**슬로건**: "Get up and running with large language models"

#### 개요
- **타겟**: 개발자, 파워유저
- **인터페이스**: CLI 중심 (터미널)
- **핵심 강점**: 단순함, 빠른 설치, 거대한 커뮤니티
- **지원 모델**: Llama, Gemma, Qwen, DeepSeek 등 100+

#### 장점
| 항목 | 설명 |
|------|------|
| 설치 간편 | `curl -fsSL https://ollama.com/install.sh \| sh` 한 줄 |
| 모델 라이브러리 | 100+ 모델 원클릭 다운로드 |
| REST API | `localhost:11434` 표준 API |
| 커뮤니티 | Discord, Reddit 활발 |
| 성능 | 최적화 잘 됨 (llama.cpp 기반) |

#### 단점
| 항목 | 설명 | OpenKlaw 기회 |
|------|------|--------------|
| CLI 전용 | 일반인 진입장벽 높음 | GUI 앱 제공 |
| 영어 중심 | 한국어 UX 없음 | 한국어 UI/UX |
| 문서 처리 없음 | 채팅만 가능 | HWP/PDF 지원 |
| 업무 특화 없음 | 범용 도구 | 사무직 특화 기능 |

#### 기술 스택
```
- Backend: Go
- LLM Engine: llama.cpp
- 배포: 바이너리 + 모델 파일
- API: REST (OpenAI 호환)
```

### 2.2 LM Studio

**슬로건**: "Local AI on your computer"

#### 개요
- **타겟**: 개발자 + 일반 유저 (하이브리드)
- **인터페이스**: Electron GUI + CLI (llmster)
- **핵심 강점**: 예쁜 UI, 모델 탐색 기능
- **최근**: Claude Code 연동 지원 발표

#### 장점
| 항목 | 설명 |
|------|------|
| GUI | 직관적인 채팅 인터페이스 |
| 모델 검색 | HuggingFace 모델 검색/다운로드 |
| 다중 모델 | 여러 모델 동시 관리 |
| 서버 모드 | 로컬 API 서버 기능 |
| 크로스플랫폼 | Windows, Mac, Linux |

#### 단점
| 항목 | 설명 | OpenKlaw 기회 |
|------|------|--------------|
| 무거움 | Electron 기반, RAM 많이 사용 | Tauri (경량) |
| 폐쇄적 | 핵심 코드 비공개 | 오픈소스 |
| 영어 전용 | 한국어 미지원 | 한국어 특화 |
| 범용 도구 | 특화 기능 없음 | HWP + 이메일 |
| 설치 크기 | 수백 MB | 50MB 이하 목표 |

#### 기술 스택
```
- Frontend: Electron
- Backend: TypeScript/Python
- LLM Engine: MLX (Mac), CUDA (Windows)
- 새로운 CLI: llmster (서버용)
```

### 2.3 Jan.ai

**슬로건**: "Open-source ChatGPT replacement"

#### 개요
- **타겟**: 일반 사용자 (ChatGPT 대체재)
- **인터페이스**: Tauri GUI
- **핵심 강점**: ChatGPT 스러운 UX, Connectors
- **다운로드**: 4백만+ (2026년 기준)

#### 장점
| 항목 | 설명 |
|------|------|
| ChatGPT UX | 익숙한 채팅 인터페이스 |
| Connectors | Gmail, Notion, Slack 연동 |
| Memory (예정) | 사용자 컨텍스트 기억 |
| MCP 지원 | Agentic 기능 |
| 오픈소스 | Apache 2.0 |

#### 단점
| 항목 | 설명 | OpenKlaw 기회 |
|------|------|--------------|
| 글로벌 타겟 | 한국 시장 무관심 | 한국 특화 |
| HWP 미지원 | 서양 문서만 | HWP 파서 |
| 한국어 UX 없음 | 영어 인터페이스 | 한국어 100% |
| 무거움 | 많은 기능 = 복잡 | 핵심 기능 집중 |
| Connector 복잡 | 설정 어려움 | 간편 설치 |

#### 기술 스택
```
- Frontend: Tauri (Rust + React)
- LLM Engine: llama.cpp (via extensions)
- 확장성: Extensions 시스템
```

**중요**: Jan.ai도 Tauri 사용 → OpenKlaw 기술 스택 검증됨

### 2.4 기업 제공 AI

#### 삼성 가우스 (Samsung Gauss)
| 항목 | 설명 |
|------|------|
| 제공 대상 | 삼성그룹 임직원 |
| 기능 | 코드 생성, 문서 요약, 번역 |
| 제한 | 외부 유출 금지, 일부 부서만 |
| 문제점 | 접근권 제한, 기능 부족 |

#### 현대차 H-GPT
| 항목 | 설명 |
|------|------|
| 제공 대상 | 현대차그룹 임직원 |
| 기능 | 내부 문서 검색, QA |
| 제한 | 파일럿 단계, 일부만 사용 |
| 문제점 | 사용 승인 필요, 대기 시간 |

#### 공통 문제점
1. **접근 제한**: 모든 직원이 사용 불가
2. **기능 제한**: 보안상 기능 축소
3. **속도**: 서버 기반, 피크 시 느림
4. **커스터마이징 불가**: 정해진 기능만

## 5. SWOT 분석 (OpenKlaw)

### Strengths (강점)
- 한국 시장 유일한 로컬 HWP AI
- Tauri 기반 경량 앱 (검증된 기술)
- 오픈소스 (신뢰성)
- 명확한 타겟 (한국 사무직)

### Weaknesses (약점)
- 신규 프로젝트 (인지도 없음)
- 1인 개발 (리소스 제한)
- 커뮤니티 없음 (초기)

### Opportunities (기회)
- 대기업/공공기관 AI 차단 = 수요 있음
- 경쟁사 한국 무관심 = 시장 비어있음
- MS Copilot 비용 부담 = 무료 대안 수요

### Threats (위협)
- 기업 자체 AI 확대 가능성
- OpenAI 오프라인 모드 출시 가능성
- 규제 변화 (AI 사용 정책)
