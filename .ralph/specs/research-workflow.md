### 1.2 반복되는 업무 패턴

#### 매일 반복
1. **이메일 트리아지**: 30-50개 메일 확인, 중요도 분류
2. **상황 보고**: 팀장/임원에게 진행 상황 업데이트
3. **자료 요청 대응**: 타 부서 데이터/자료 요청 처리
4. **일정 조율**: 미팅 시간 조정, 회의실 예약

#### 매주 반복
1. **주간 보고서**: 주간 성과 및 계획 정리
2. **정기 회의**: 팀 회의, 부서 회의 참석
3. **자료 백업/정리**: 문서 폴더 정리

#### 매월 반복
1. **월간 보고서**: KPI 정리, 성과 분석
2. **예산 검토**: 비용 집행 현황 정리
3. **다음 달 계획**: 일정 및 목표 수립

## 2. 시간 소요가 큰 업무 TOP 5

### 🥇 1위: 보고서/문서 작성 (주 8-12시간)
- **상세**: 주간보고, 월간보고, 회의록, 기획안, 제안서
- **Pain Point**: 
  - 포맷 맞추기에 시간 낭비
  - HWP ↔ Word 변환 문제
  - 이전 문서 찾아서 복붙하는 반복 작업
- **AI 기회**: 템플릿 자동 생성, 문서 요약, 문서 형식 변환

### 🥈 2위: 이메일 작성/답변 (주 5-8시간)
- **상세**: 업무 요청, 일정 조율, 상황 공유, 공식 요청
- **Pain Point**:
  - 격식 맞추기 (높임말, 호칭)
  - 비슷한 내용 반복 작성
  - 영어 이메일 (해외 파트너)
- **AI 기회**: 이메일 초안 생성, 톤 조정, 번역

### 🥉 3위: 자료 검색/정리 (주 4-6시간)
- **상세**: 사내 문서 검색, 과거 데이터 찾기, 정보 취합
- **Pain Point**:
  - 사내 시스템이 분산 (그룹웨어, 파일서버, 메일)
  - 검색 기능 부실
  - 필요한 정보가 HWP/PDF에 묻혀있음
- **AI 기회**: 문서 인덱싱, 의미 검색, 자동 요약

### 4위: 회의 참석/후속 처리 (주 4-5시간)
- **상세**: 회의 참석, 회의록 작성, 액션 아이템 정리
- **Pain Point**:
  - 회의록 작성에 회의 시간만큼 소요
  - 누가 무슨 말 했는지 기억 안남
  - 후속 조치 누락
- **AI 기회**: 자동 회의록, 액션 아이템 추출

### 5위: 엑셀 데이터 작업 (주 3-5시간)
- **상세**: 데이터 취합, 포맷 정리, 간단한 분석
- **Pain Point**:
  - 수식 복잡해지면 막힘
  - 여러 파일 데이터 합치기
  - 피벗테이블/차트 매번 새로 만들기
- **AI 기회**: 엑셀 수식 생성, 데이터 분석 보조

### 3.3 OpenKlaw 특화 기회

#### HWP 지원의 가치
- **시장 현황**: 정부기관/공공기관 90%+, 대기업 60%+ HWP 사용
- **경쟁 제품**: Ollama, LM Studio, Jan.ai 모두 HWP 미지원
- **OpenKlaw 기회**: 유일한 로컬 AI + HWP 지원 = 블루오션

#### 한국어 특화
- **비즈니스 한국어**: 존댓말, 호칭, 격식
- **기업 문화 반영**: 결재선, 보고 라인 인식
- **문서 양식**: 한국 기업 표준 양식 이해

## 4. 타겟 사용자 세그먼트

### 4.1 프라이머리 타겟: 대기업/공공기관 사무직

| 특성 | 설명 |
|------|------|
| 규모 | 1,000명+ 기업, 정부기관 |
| AI 제약 | ChatGPT/Claude 접속 차단 (90%+) |
| 장비 | 회사 지급 노트북 (i5, 16GB RAM, 내장GPU) |
| 문서 | HWP 필수, 간혹 Word |
| Pain | 반복 업무 多, 야근 잦음, 효율화 욕구 높음 |

### 4.2 세컨더리 타겟: 중소기업 사무직

| 특성 | 설명 |
|------|------|
| 규모 | 50-500명 기업 |
| AI 제약 | 일부 제한 (50% 정도) |
| 장비 | 다양 (개인 노트북 혼용) |
| 문서 | HWP + Word 혼용 |
| Pain | 인력 부족, 멀티태스킹 필수 |

### 4.3 페르소나

#### 김과장 (35세, 대기업 기획팀)
- 삼성전자 기획팀 과장, 입사 8년차
- 매일 보고서 3-4건, 이메일 50건+
- ChatGPT 써보고 싶지만 보안팀이 막음
- "개인정보 안 나가는 AI가 있으면 바로 쓸텐데..."

#### 이대리 (29세, 공공기관)
- 국토교통부 산하기관, 입사 3년차
- HWP 문서 하루종일, 엑셀 데이터 정리
- 외부 클라우드 서비스 일절 사용 불가
- "내 노트북에서만 돌아가는 AI가 있다고?"

## 5. 주요 발견사항

### 5.1 로컬 AI 수요 근거

1. **보안 이슈가 최대 관문**
   - 대기업 90%+, 공공기관 100% 외부 AI 차단
   - "데이터 유출 우려"가 공식적 이유
   - 개인적으로 우회 사용하는 직원 다수 (리스크)

2. **생산성 도구 갈증**
   - MS Copilot: 비싸고 (월 3만원+), 온라인 필수
   - 회사 제공 AI: 극소수, 있어도 기능 제한
   - "합법적으로 쓸 수 있는 AI"에 대한 니즈

3. **한국어 + HWP = 미개척 시장**
   - 글로벌 도구들 한국 시장 무시
   - HWP 지원하는 AI 도구 전무
   - "한국 직장인 전용"이라는 포지셔닝 가능

### 5.2 도입 시 예상 허들

1. **IT 부서 승인**: "로컬이라도 AI는 AI"
2. **설치 복잡성**: 비개발자가 설치할 수 있어야 함
3. **성능 우려**: "로컬이면 느리지 않나?"
4. **지속적 지원**: "회사가 없어지면?"

## 6. 경쟁 제품 분석

### 6.1 Ollama

| 항목 | 내용 |
|------|------|
| **강점** | 쉬운 설치 (`brew install`), 다양한 모델 지원, 빠른 업데이트 |
| **약점** | CLI 전용, GUI 없음, 사무용 특화 기능 X |
| **타겟** | 개발자, AI 연구자 |
| **가격** | 무료 (오픈소스) |
| **한국어** | 모델 의존 (Qwen, EXAONE 지원) |
| **HWP** | ❌ 미지원 |

**OpenKlaw 차별점**: GUI + 사무용 스킬 + HWP 지원

### 6.2 LM Studio

| 항목 | 내용 |
|------|------|
| **강점** | 깔끔한 GUI, 모델 다운로드 통합, 채팅 인터페이스 |
| **약점** | 무거움 (500MB+), 도구 통합 없음, 스킬 없음 |
| **타겟** | AI 입문자, 개발자 |
| **가격** | 무료 |

| **HWP** | ❌ 미지원 |

## 7. 결론 및 권고

### MVP 핵심 기능 (Must Have)
1. ✅ 기본 채팅 (한국어 최적화)
2. ✅ HWP 문서 요약
3. ✅ 이메일 작성 보조
4. ✅ 원클릭 설치 (Windows exe / macOS npm)

### 차별화 포인트
1. **유일한 로컬 HWP 지원 AI** - 경쟁사 전무
2. **한국 비즈니스 한국어 특화** - 존댓말, 격식, 호칭
3. **5분 안에 시작 가능한 간편 설치** - 비개발자도 OK
4. **경량 설계** - 50MB 이하 (LLM 제외)

### 기술 결정
- **LLM**: Qwen2.5-3B-Instruct (기본), EXAONE-3.5 (옵션)
- **Runtime**: Ollama (안정적, 쉬운 설치)
- **Desktop**: Tauri (경량, 크로스플랫폼)
- **UI**: SvelteKit (빠름, 가벼움)
